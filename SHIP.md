# Hack Club Ship — submission checklist

Use this after you push to GitHub.

## Before you ship

- [ ] Install [Hackatime](https://hackatime.hackclub.com/) and log **≥ 1 hour** on this project (more is better for a hardware build).
- [ ] Write at least **one devlog** on the Ship site (build progress, wiring, bugs you fixed).
- [ ] Complete **peer voting** (Ship shows how many votes you still need).
- [ ] Add **photos** under `assets/photos/` (desk unit, plant node, dashboard) and embed them in `README.md`.
- [ ] Export **`assets/banner.png`** from `assets/banner.svg` (1200×630) and upload as your project **banner** on Ship (not from a devlog image).

## Links to enter on Ship

| Field | Value |
|-------|--------|
| **Repository** | `https://github.com/MatsDevOfficial/arcade` |
| **Raw README** | `https://raw.githubusercontent.com/MatsDevOfficial/arcade/main/README.md` |
| **Demo** | Your YouTube/Loom URL (see `docs/demo-video-script.md`) |

The raw README link must be the **raw.githubusercontent.com** URL, not the github.com HTML page.

## Description (copy-paste starter, ≥ 30 chars)

> Smart Desk Assistant is a modular ESP32 workspace platform: an ESP32-S3 desk controller with OLED, microphone, and buttons, a wireless ESP32-C3 plant node (soil + temperature), and an Electron dashboard over USB serial.

## Double-dipping

Do **not** submit the same project hours to another YSWS program (e.g. Blueprint) unless that program allows it. Check each program’s rules.
